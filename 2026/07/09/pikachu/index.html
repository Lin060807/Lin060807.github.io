宽字节注入

通过编码绕过后端代码的防御措施，例如正则过滤和转义函数转义。

使用场景：网站用 addslashes() 防御 SQL 注入，会把你输入的单引号 ‘ 转义成 '，单引号失效，没法闭合 SQL 语句注入。

客户端采用GBK编码格式，数据库对用户输入进行转义\，转义符\的编码是%5c，添加编码%df，组成%df%5c，识别成一个生僻汉字，绕过转义符让‘逃逸，反斜杠\直接被 “吃掉、消失”，最后只剩单独的 ‘，成功闭合 SQL 语句，注入生效

RCE

远程系统命令执行

后端直接把用户可控输入拼接进系统命令，没有过滤 / 转义，攻击者可以插入命令分隔符，额外执行任意系统指令,可以让攻击者直接向后台服务器远程注入操作系统命令或者代码，从而控制后台系统。

应用系统从设计上需要给用户提供指定的远程命令操作的接口，会给用户一个ping操作的web界面，用户从该界面输入目标IP，提交后后台对该IP地址进行一次ping测试，返回结果。

命令分隔符

Windows / Linux 通用
; 分号：无论前命令成功失败，两条命令都执行
127.0.0.1;whoami

| 管道符：把前命令输出交给后命令，强制执行后面指令
127.0.0.1|whoami

逻辑判断符

&&：前命令执行成功，才执行后面
127.0.0.1&&whoami

||：前命令执行失败，才执行后面
127.0.0.1||whoami

Linux 独有

& 后台运行、$() 命令嵌套、反引号 嵌套执行
127.0.0.1$(whoami)

File Include

文件包含函数：include(),include_once(),require(),require_once()
文件包含的代码文件被写成了一个变量，变量由前端客户传进来，让文件包含函数去执行，造成恶意1操作

1.本地文件包含漏洞：仅能够对服务器本地的文件进行包含，通常情况攻击者会包含固定的系统配置文件，读取系统敏感信息。

2.远程文件包含漏洞：通过URL地址对远程的文件进行包含，攻击者可以传入任意的代码
